# Алгоритм объединения DWG-файлов

Этот документ описывает пошаговый процесс работы инструмента AutoBIMFusion для объединения нескольких DWG-файлов в один.

## Общий процесс

Процесс состоит из следующих этапов:
1. Выбор папки с исходными DWG-файлами.
2. Проверка каждого файла на пригодность к объединению.
3. Экспорт содержимого Layout (Paper Space) каждого файла в Model Space временного DWG.
4. Вставка полученных временных DWG в целевой файл как внешние ссылки (Xref) с их последующей привязкой (Bind).

---

## Подробные шаги алгоритма

### Шаг 1: Подготовка
- Пользователь запускает команду MERGEDWG.
- Инструмент запрашивает выбор папки через FolderSelector.
- FileEnumerator сканирует папку, исключая файлы с префиксом # и файлы размером более 15 МБ.

### Шаг 2: Итерация по файлам
- Для каждого выбранного файла:
    - Проверка через FileHelper (структура DWG, доступность файла).
    - Если проверка не пройдена: файл пропускается (MergeResult.Warn).
    - Если проверка пройдена: вызывается DwgMerger.MergeSingleFile.

### Шаг 3: Экспорт Layout (Paper Space) во временный файл
- ViewportLayoutExporter открывает исходный файл.
- Определяется основной видовой экран (Viewport) на выбранном листе (Layout):
    - Если экранов нет или их несколько, используется LayoutViewportInfo для выбора главного.
- Создается временный файл DWG.
- Всё содержимое Layout (включая объекты Paper Space) переносится в Model Space временного файла.
- Если есть видовые экраны, выполняется трансформация объектов Model Space согласно настройкам видовых экранов.
- Выполняется ModelSpaceTrimmer для удаления объектов, выходящих за границы основного видового экрана.
- Содержимое сохраняется во временный файл DWG.

### Шаг 4: Вставка во временный файл (Xref)
- DwgMerger читает границы (Extents) полученного временного DWG.
- BlockInserter создает уникальное имя блока и вставляет этот временный файл в целевой DWG как Xref.
- Позиционирование блока рассчитывается так, чтобы он не перекрывался с предыдущими вставленными блоками.
- Xref привязывается (Bind) к целевому файлу.

### Шаг 5: Завершение
- Временные файлы удаляются.
- Целевой файл сохраняется, выполняется REGENALL и ZOOM EXTENTS.

# Алгоритм работы AutoBIMFusion

**Последнее обновление:** 2026-05-02

## 1. Запуск и выбор файлов
1. Команда `MERGEDWG` блокирует повторный запуск через `SemaphoreSlim`.
2. Пользователь выбирает папку; плагин собирает все `.dwg` файлы (глубина поиска — 3 уровня).
3. Файлы сортируются в естественном порядке. Файлы >15 МБ или с префиксом `#` пропускаются.

## 2. Подготовка базы данных (In-Memory)
1. Каждый файл открывается в фоновой базе данных (`Database`).
2. Принудительно устанавливаются метрические единицы (`Millimeters`, `Metric`).
3. Находится первый лист (Layout), отличный от модели.
4. Пространство листа проецируется в пространство модели этой же базы:
   - Если видовых экранов нет: масштаб 1:100.
   - Если видовые экраны есть: выбирается основной, его масштаб ограничивается (не крупнее 1:100), остальные окна трансформируются в его систему координат.
5. Для размеров Model Space сохраняется effective multiplier рабочего clamped main VP (`1 / mainClamped.CustomScale`). Для размеров, попадающих в несколько VP, по-прежнему выбирается viewport с меньшей областью модели как источник соответствия, но значение масштаба берется из clamped main VP.
6. Лишние объекты вне рамки листа удаляются (`TrimOutside`).
7. До клонирования в целевой чертеж выполняется нормализация размеров: создаются стили `"{Style} - Scale {Multiplier}"`, очищаются `ACAD`/`DSTYLE` XData overrides, размеру назначается новый стиль, `Dimscale` и `Dimlfac` приводятся к `1.0`. Если исходный стиль уже содержит модельную высоту текста, визуальные величины не умножаются повторно.
8. Старые исходные размерные стили, с которых размеры были переназначены, удаляются только если `Database.Purge` подтверждает, что они больше не используются.

## 3. Вставка в целевой чертеж
1. Объекты из подготовленной базы клонируются в целевой чертеж через `WblockCloneObjects`.
2. Перед клонированием синхронизируются `Insunits` и `Measurement` для исключения авто-масштабирования стилей.
3. К клонированным объектам применяется смещение по оси X (с зазором 10% от габаритов).
4. Вставка не выполняет дополнительное лечение размеров: размерные стили уже подготовлены в исходной базе перед `WblockCloneObjects`, а смещение по X не меняет масштаб размеров.

## 4. Финализация
1. Копируются растровые изображения (если есть) в папку с итоговым файлом.
2. Выполняется оптимизация базы (`Purge`) до 5 проходов.
3. Итоговый файл сохраняется в формате AutoCAD 2018 (AC1032).
4. Выполняется `REGENALL` и `ZOOM EXTENTS`.

## 5. Управление ресурсами
- Все транзакции, базы данных и блокировки документов обернуты в `using`.
- Исключения AutoCAD API перехватываются и логируются без прерывания общего процесса.
- Временные файлы на диск не записываются (обработка идет в памяти).

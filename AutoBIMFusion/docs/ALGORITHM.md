# Алгоритм работы AutoBIMFusion

**Последнее обновление:** 2026-05-04

## 1. Запуск и выбор файлов

1. Команда `MERGEDWG` блокирует повторный запуск через `SemaphoreSlim`.
2. Пользователь выбирает папку через `UiDialogService.TrySelectFolder`; плагин собирает все `.dwg` файлы (глубина поиска — 3 уровня).
3. Файлы сортируются в естественном порядке (`WindowsNaturalComparer`). Файлы > 15 МБ или с префиксом `#` пропускаются.
4. Каждый файл предварительно проверяется на доступность (`FileUtil.TryValidateDwg`).

## 2. Подготовка базы данных (In-Memory)

*Реализовано в `ViewportLayoutExporter.PrepareDatabaseForMerge`.*

1. Файл открывается в фоновой базе данных (`new Database(false, true)`) — без записи на диск.
2. Принудительно устанавливаются метрические единицы (`Insunits = Millimeters`, `Measurement = Metric`).
3. Находится первый лист (Layout) с наименьшим `TabOrder`, отличный от Model Space.
4. Пространство листа проецируется в Model Space той же базы:
   - **Без видовых экранов:** масштаб 1:100, содержимое Paper Space трансформируется в модель.
   - **С видовыми экранами:** выбирается главный VP (максимальный `CoverageScore = PaperArea / CustomScale`), его масштаб ограничивается сверху (не крупнее 1:100). Вспомогательные VP трансформируются в систему координат главного VP.
5. Объекты вне рамки листа удаляются (`ModelSpaceTrimmer.TrimOutside`).
6. Нормализация размерных стилей (`DimensionStyleNormalizer`):
   - Для каждого размера вычисляется `effectiveMultiplier = 1 / clampedMainScale`.
   - Создаётся или переиспользуется стиль вида `"{Стиль} - Scale {Множитель}"`.
   - Очищаются per-entity XData overrides (`ACAD`/`DSTYLE`).
   - Размеру назначается нормализованный стиль; `Dimscale` и `Dimlfac` = `1.0`.
   - Если исходный стиль уже содержит модельную высоту текста (`Dimtxt > 20`), визуальные величины не умножаются.
   - Замещённые старые стили удаляются только если `Database.Purge` подтверждает их неиспользование.

## 3. Вставка в целевой чертёж

*Реализовано в `BlockInserter.InsertNativeObjects`.*

1. Синхронизируются `Insunits` и `Measurement` между базами (`ExtentsUtils.SyncUnits`).
2. Объекты из подготовленной базы клонируются через `WblockCloneObjects` с `DuplicateRecordCloning.Ignore`.
3. К каждому клону применяется смещение по оси X с зазором 10% от максимального габарита (`CalcInsertionPoint`).
4. Сохраняется порядок прорисовки через `DrawOrderPreserver`.

## 4. Финализация

1. Растровые изображения копируются в папку итогового файла; пути `RasterImageDef` обновляются на относительные (`RasterImagePathFixer`).
2. Диагностический снимок стилей записывается в лог (`DimensionStyleDiagnosticUtils.LogStyleSnapshot`).
3. Выполняется оптимизация базы (`DwgOptimizer.Optimize`) — до 5 проходов `Purge`.
4. Итоговый файл сохраняется в формате AutoCAD 2018 (`DwgVersion.AC1032`).
5. Выполняется `REGENALL` и `ZOOM EXTENTS`.

## 5. Управление ресурсами

- Все транзакции, базы и блокировки документов обёрнуты в `using` — гарантированное освобождение.
- Системные переменные AutoCAD (FILEDIA, EXPERT и др.) восстанавливаются через `AcadWarningSuppressScope` / `AcadUnitScalingOverrideScope` (RAII-паттерн).
- Обработка выполняется полностью в памяти — временные файлы на диск не пишутся.
- Исключения AutoCAD API перехватываются и логируются без прерывания пакетной обработки.
